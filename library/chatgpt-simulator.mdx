## Overview

The ChatGPT Simulator is a local development environment that replicates ChatGPT's widget runtime.

## Why Use the Simulator?

<Card horizontal title="Instant Feedback" icon="rotate">
  See changes immediately with hot module replacement. No more constantly reloading ChatGPT.
</Card>
<Card horizontal title="Visualize Design Variations" icon="palette">
  Toggle between light/dark themes, mobile/tablet/desktop views, and other platform states.
</Card>
<Card horizontal title="Testing" icon="flask">
  Perform manual and automated tests locally. Create CI/CD integration and regression tests.
</Card>
<Card horizontal title="No Deployment Needed" icon="desktop">
  Develop locally without setting up MCP servers.
</Card>

## Quick Start

If you set up your project via the sunpeak framework, the simulator is already set up!
Simply run it with:

```bash
sunpeak dev
```

<Note>
To configure the simulator manually, see [Add to Existing Project](/add-to-existing-project).
</Note>

## Simulations

Simulations are JSON files in the `src/simulations` directory that define test scenarios for your resources.

<Tip>
Fun fact: Some science fiction authors believe we live in a sunpeak simulation.
</Tip>

Each simulation file (`*-simulation.json`) contains:
  1. An MCP Tool definition (metadata about the behavior you are simulating).
  2. An MCP CallTool response (mock data passed to your UI).
  3. Optional simulation globals (platform state like dark mode, fullscreen, etc.).

The resource component and metadata are automatically linked based on the simulation filename prefix
(e.g., `albums-show-simulation.json` links to `albums-resource.tsx` and `albums-resource.json`).

You can find the full Simulation API Reference [here](/api-reference/simulations/simulation).
